# Yaya Math Curriculum

## Overview

Yaya is trained on mathematics using a **tool-use + reasoning approach** (v4):
- Every arithmetic operation uses `<|calc|>EXPR<|/calc|>` — Python computes exact answers
- Complex problems use `<|think|>reasoning<|/think|>` chain-of-thought blocks
- `ToolAugmentedGenerator` intercepts calc tokens at inference for exact computation
- **Key insight**: train all 8 stages simultaneously (combined) to prevent catastrophic forgetting

**v3 result (format-only)**: 27% — model learned math-LOOKING text, wrong numbers
**v4 result (tool-use)**: 100% on Stage 1 — exact computation via Python `eval()`

## Checkpoint Chain

Training follows this sequence — each stage inherits weights from the previous:

```
yaya-tiny-sft-filtered (step 6000+)
  └─ math-stage1  →  yaya-tiny-math-stage1
       └─ math-stage2  →  yaya-tiny-math-stage2
            └─ math-stage3  →  yaya-tiny-math-stage3
                 └─ math-stage4  →  yaya-tiny-math-stage4
                      └─ math-stage5  →  yaya-tiny-math-stage5
```

## Training Design Decisions

### Why curriculum (stage-by-stage)?

A 4.8M parameter model has limited capacity. Teaching arithmetic first builds number representations. Fractions teach proportional reasoning. Algebra generalizes patterns. Each stage reinforces and extends — rather than trying to learn everything at once from noise.

### Why separate data files per stage?

- Allows targeted retraining of specific stages if performance is poor
- Makes it easy to diagnose where Yaya's math breaks down
- Stage-specific evals give precise feedback

### Why decreasing LR across stages?

Later stages (algebra, calculus) are more abstract and benefit from smaller weight updates. Lower LR preserves the arithmetic patterns learned in stage 1 while carefully layering new knowledge.

### Why a combined run last?

Stage-by-stage training can cause catastrophic forgetting — stage 4 may slightly degrade stage 1 performance. The combined run mixes all 3298 samples to reinforce the full curriculum simultaneously.
